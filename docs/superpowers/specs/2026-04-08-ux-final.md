# Poke Pal v0.7.0 -- Final UX Specification

**Date:** 2026-04-08
**Status:** FINAL (with amendments) -- supersedes all previous UX specs

## Late Amendments (from founder, OVERRIDE any conflicting spec content)

### Amendment A: Teams page CopyBar position
The "Find your team in-game" CopyBar should appear RIGHT AFTER the team slots, not buried below other content. Follow the same pattern as league pages: team selection → CopyBar → details/analysis.

### Amendment B: Kill the picker sheet popup
Remove the TeamSlotPicker bottom sheet entirely. Instead, when a slot is empty, show inline suggestion pills directly below that slot:
- Meta picks first as tappable chips
- Then a type hint: "or any Dragon, Fairy type" telling players what types to look for if they don't have the exact Pokemon
- Suggestions are contextual — they update based on what's already in other slots (from analyzeTeam suggestions)

Layout per empty slot:
```
Safe Swap: [tap to add]
  Try: [Altaria] [Azumarill] [Steelix]  or any Dragon, Fairy
```

Each slot gets its own suggestions. No popup, no sheet, no separate screen. Everything inline.

### Amendment C: Type hints for team building
When showing suggestions, include which types would fill the team's gaps. This helps players who don't have the exact meta picks but have other Pokemon of the right type. Format: "or any [Type], [Type]" after the suggestion chips.
**Supersedes:** `ux-consistency.md`, `ux-overhaul.md`, `ux-audit.md`
**Target device:** iPhone 375px (primary), any mobile browser

## 1. Global Rules

These rules apply across every page. Violations are bugs.

### 1.1 CopyBar Labels

Every `CopyBar` instance MUST receive a `label` prop. The label renders as `text-xs text-muted-foreground` above the bar (not inside it).

| Context | `label` value |
|---------|---------------|
| Counter page (main) | `"Counters search string"` |
| Counter page (shadow) | `"Shadow counters search string"` |
| League page, no team | `"Search for meta Pokemon"` |
| League page, 1+ team members | `"Find your team in-game"` |
| Teams page | `"Your team search string"` |
| Inline team section | REMOVED -- the league page CopyBar handles it |

### 1.2 Single Contextual CopyBar on League Pages

Currently there are two copy bars on league pages (meta search string + team search string inside inline-team-section). Replace with ONE `CopyBar` at the page level that changes based on team state:

- **No team selected:** shows meta search string, label `"Search for meta Pokemon"`
- **1+ Pokemon in team:** switches to team search string, label `"Find your team in-game"`

The inline team section no longer contains its own CopyBar.

### 1.3 Button Labels

| Current | New |
|---------|-----|
| `"Full Analysis -->"` | `"Team Builder -->"` |
| `"Build a Team" (static)` | State-dependent (see Home Page) |

Arrow `-->` suffix = navigation to another page. Action buttons (Clear, Copy, Share) do NOT get arrows.

### 1.4 Clear Button Design

All "Clear" actions use a shared `ClearButton` component. Rules:
- Always visible when there is something to clear
- Positioned at the right edge of its section header
- No confirmation dialog -- teams are local state and easy to rebuild
- After clearing: team array becomes `[]`, localStorage key is removed via `clearTeam(leagueId)`

Visual: `rounded-full border px-2.5 py-1 text-xs font-medium text-muted-foreground hover:text-foreground hover:bg-muted` with `min-h-8` tap target. Text: "Clear".

### 1.5 Pokemon Display: Two Components Only

| Component | Purpose | Where used |
|-----------|---------|------------|
| `PokemonListItem` | Information-dense row in lists | Counter results, league tier accordions, team builder meta picks |
| `PokemonChip` | Compact pill for team compositions | Inline team section, home page team preview |

No other Pokemon display patterns should exist.

### 1.6 Search Available on Every Tab

`SearchInput` appears at the top of every main page: Home, Leagues landing, League detail, and Teams. Behavior varies by context:

| Page | Search behavior on select |
|------|--------------------------|
| Home (`/`) | Navigate to `/counter/[pokemon]` |
| Leagues landing (`/leagues`) | Navigate to `/counter/[pokemon]` |
| League detail (`/league/[id]`) | Add Pokemon to team (calls `handleAddToTeam`) |
| Teams (`/teams`) | Add to next empty slot (calls `handlePokemonSelect`) |

The `SearchInput` component gains a new prop: `mode: "navigate" | "add"` and a callback `onSelect?: (pokemonId: string) => void`. When `mode="navigate"` (default), it pushes to `/counter/[id]`. When `mode="add"`, it calls `onSelect` instead.

### 1.7 Team Rating (S/A/B/C)

Calculate from average tier of selected Pokemon. Tier score mapping: S=4, A=3, B=2, C=1.

| Average score | Rating |
|---------------|--------|
| 3.5+ | S |
| 2.5 -- 3.49 | A |
| 1.5 -- 2.49 | B |
| Below 1.5 | C |

Display: a colored letter using the same tier colors as the tier accordion, shown next to "Your Team" in the inline team section header.

Tier color mapping (from `meta-pokemon-card.tsx`):
- S: `text-yellow-600 dark:text-yellow-400`
- A: `text-blue-600 dark:text-blue-400`
- B: `text-green-600 dark:text-green-400`
- C: `text-muted-foreground`

### 1.8 Tier List Highlights Recommended Pokemon

When 1+ Pokemon are in the team, the tier list cards for RECOMMENDED Pokemon (from `analyzeTeam().suggestions`) get a visual "Rec" tag. This replaces the "Try:" suggestion chips in the inline team section entirely.

Implementation:
- Pass `analysis.suggestions` pokemonIds to the `TierAccordion` as a `recommendedIds?: string[]` prop
- `PokemonListItem` gains `recommended?: boolean` prop
- When `recommended` is true, show a small `"Rec"` badge: `rounded-full bg-emerald-100 dark:bg-emerald-900 px-1.5 py-0.5 text-[10px] font-semibold text-emerald-700 dark:text-emerald-300`
- Sort recommended Pokemon to the top within each tier group
- The "Try:" line in inline-team-section is removed

## 4. State Flow Diagram

### 4.1 Storage Format

Key: `poke-pal:team:[leagueId]`
Value: `JSON.stringify(string[])` -- array of pokemon IDs

### 4.2 SAVE Rules

Whenever a Pokemon is added or removed, on ANY page:
- `saveTeam(leagueId, currentTeamIds)` fires in a `useEffect` watching the team array
- Targets the CURRENT league's key only

### 4.3 LOAD Rules

| Context | When | Source | Behavior |
|---------|------|--------|----------|
| Teams page, initial mount | URL has `?p=` | URL params | Use URL pokemon, ignore localStorage |
| Teams page, initial mount | URL has NO `?p=` | localStorage for URL league | Load saved team |
| Teams page, league switch | User taps different league | N/A | Save current team under old league key, CLEAR displayed slots, do NOT load new league's saved team |
| League detail, mount | Always | localStorage for this league | Load saved team |
| Home page, mount | Always | All `poke-pal:team:*` keys | Show preview for each league with a saved team |

### 4.4 Teams Page State Fix

The current bug: `useEffect` depends on `[league]`, so it reloads from localStorage every time the league changes, overwriting user state.

Fix: split mount-only initialization from league-change handling.

```typescript
// Mount-only: load from URL or localStorage ONCE
const mountedRef = useRef(false);
useEffect(() => {
  if (mountedRef.current) return;
  mountedRef.current = true;
  if (initialPokemon.length === 0 && team.every((s) => s === null)) {
    const stored = loadTeam(league);
    if (stored.length > 0) {
      setTeam([
        stored[0] ? pokemonToSlot(stored[0]) : null,
        stored[1] ? pokemonToSlot(stored[1]) : null,
        stored[2] ? pokemonToSlot(stored[2]) : null,
      ] as [TeamSlot, TeamSlot, TeamSlot]);
    }
  }
}, []); // mount only

// League switch handler
const handleLeagueChange = useCallback((newLeague: LeagueId) => {
  // 1. Save current team under current league
  const currentIds = team
    .filter((s): s is NonNullable<TeamSlot> => s !== null)
    .map((s) => s.pokemonId);
  saveTeam(league, currentIds);
  
  // 2. Clear all slots
  setTeam([null, null, null]);
  
  // 3. Switch league
  setLeague(newLeague);
  
  // Do NOT load from localStorage for the new league
}, [team, league]);
```

### 4.5 CLEAR Rules

- Explicit `ClearButton` tap only
- Switching leagues does NOT clear localStorage (old league's team stays saved)
- `clearTeam(leagueId)` removes the localStorage key entirely

### 4.6 New Helper: `getAllSavedTeams()`

Add to `src/lib/team-storage.ts`:

```typescript
export function getAllSavedTeams(): Array<{ leagueId: string; pokemonIds: string[] }> {
  const teams: Array<{ leagueId: string; pokemonIds: string[] }> = [];
  try {
    for (let i = 0; i < localStorage.length; i++) {
      const key = localStorage.key(i);
      if (key?.startsWith(STORAGE_KEY_PREFIX)) {
        const leagueId = key.slice(STORAGE_KEY_PREFIX.length);
        const val = localStorage.getItem(key);
        if (val) {
          const parsed = JSON.parse(val);
          if (Array.isArray(parsed) && parsed.length > 0) {
            teams.push({ leagueId, pokemonIds: parsed });
          }
        }
      }
    }
  } catch {}
  return teams;
}
```

### 4.7 Team Rating Calculation

Add to `src/lib/team-analysis.ts` or inline:

```typescript
const TIER_SCORES: Record<string, number> = { S: 4, A: 3, B: 2, C: 1 };

export function calculateTeamRating(
  teamIds: string[],
  leagueMeta: MetaPokemon[]
): "S" | "A" | "B" | "C" | null {
  if (teamIds.length === 0) return null;
  const scores = teamIds.map((id) => {
    const meta = leagueMeta.find((m) => m.pokemonId === id);
    return meta ? (TIER_SCORES[meta.tier] ?? 1) : 1;
  });
  const avg = scores.reduce((a, b) => a + b, 0) / scores.length;
  if (avg >= 3.5) return "S";
  if (avg >= 2.5) return "A";
  if (avg >= 1.5) return "B";
  return "C";
}
```
