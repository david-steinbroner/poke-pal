Last updated: 2026-04-08 (end of marathon session)

## Shipped (v1.0.1) -- 2026-04-08

Everything live at https://poke-pal.pages.dev:

### Counter Search
- Type any Pokemon, get top 5 counters + budget picks + copyable GO search string
- Type effectiveness badges (Weak to / Resists with 2x multipliers)
- Single "Copy Counters Search String" button
- BackButton with history fallback
- 119 Pokemon with real GO stats, moves, base stats

### League Meta
- Great League, Ultra League, Master League, Fantasy Cup with S/A/B/C tier rankings
- Combined league search strings -- one-tap copy of all meta Pokemon names + CP cap
- S tier expanded by default, other tiers collapsed
- Leagues landing page (`/leagues`) -- Live Now + Coming Up sections

### Team Builder
- Pick 3 Pokemon for any league, see coverage analysis, meta threats
- In-place team building on league pages -- + button on every meta card, inline team section with smart suggestions
- Cup selector on Teams page -- one team per league, navigate between them
- League tabs always visible, shows saved team status per league
- Role-based analysis (lead / safe-swap / closer) -- subtle, informational
- Persistent teams via localStorage
- Shareable team URLs via query params

### Copy System
- CopyButton + DualCopyButtons replaced CopyBar everywhere
- Copy toast: "Copied -- paste in Pokemon GO search"
- Search string hidden from user -- just copy buttons

### Navigation
- Bottom nav: Home | Leagues | Teams on all pages
- League tab memory -- remembers last viewed league
- Text-only nav (no icons)

### Design System
- TypeBadge component with multiple variants
- StatusPill, PokemonChip components
- 3-tier pill sizing system
- Font accessibility: text-[13px] minimum (was text-xs)

### Infrastructure
- OG meta tags for link previews
- Static export: 129 pages, edge-cached on Cloudflare Pages
- 33 tests passing, TypeScript strict mode, zero errors
- iOS mobile support: clipboard fallback, 44px touch targets, active state fixes
- Auto-deploys on push to GitHub

## Known Issues (prioritized)

1. **Stale raid/league data** -- league JSON files are manually maintained. No update mechanism. Fantasy Cup will expire and there is no way to rotate it out without a code change.
2. **League barrel export** -- adding a new league currently requires touching 5+ files (JSON, constants, types, pages, nav). Needs a barrel export or registry pattern.
3. **OG images** -- meta tags exist but no actual images. Link previews on Discord/Reddit/Twitter show no thumbnail.
4. **Empty state UX** -- new users on home page see Current Raids and What's Live but no guidance. Needs a first-visit experience.
5. **Counter to team builder flow** -- after viewing counters for a Pokemon, there is no way to start a team with those counters. The two features are disconnected.
6. **Data freshness indicator** -- users have no way to know if league/raid data is current or stale.

## Future Sprints (when unpaused)

### Data Scaling (prerequisite for full Pokedex)
- **Slim search index** -- create `pokemon-search-index.json` with just `{id, name}` pairs (~20KB for 1000 Pokemon). SearchInput uses this instead of the full 83KB+ pokemon.json. Counter pages continue using full data at build time (server only). MUST do this before expanding beyond ~200 Pokemon.
- **Full Pokedex** -- expand from 119 to ~1000 Pokemon with GO-relevant stats/moves. ~1000 static counter pages. Cloudflare Pages handles this fine, build time increases to ~60s.

### Features
- Active cup automation -- auto-rotate with GBL schedule instead of manual league JSON updates
- More cups from GBL schedule -- Spring Cup, Jungle Cup, etc.
- Better counter scoring -- factor in STAB, bulk, energy generation, shield pressure (currently ATK-only)
- SEO pages for popular teams -- "Best Great League Team April 2026" style content
- Collection import -- let users import their Pokemon storage for personalized recommendations
- Service worker / offline PWA -- cache static pages for GO events without signal
- Reddit/Discord launch -- gif/video of copy-paste flow, timed to GBL season changes
- Team Rocket tab -- separate feature, needs data pipeline first
