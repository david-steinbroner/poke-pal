# Poke Pal Roadmap

Last updated: 2026-04-08

---

## Shipped (v0.1.0) — 2026-04-08

Everything live at https://poke-pal.pages.dev:

- **Counter search** — type any Pokemon, get top 5 counters + budget picks + copyable GO search string
- **119 Pokemon** with real GO stats, moves, base stats (Charizard, Snorlax, all Eeveelutions, legendaries, starters, 60+ more)
- **League meta** — Great League, Ultra League, Master League, Fantasy Cup with S/A/B/C tier rankings
- **Combined league search strings** — one-tap copy of all meta Pokemon names + CP cap for pasting in GO
- **Team Builder** (`/teams`) — pick 3 Pokemon for any league, see 18-type coverage chart, defensive weaknesses, meta threats, swap suggestions
- **In-place team building on league pages** — + button on every meta card, floating bar with smart suggestions, coverage score when full
- **Fantasy Cup: Great League Edition** — Dragon/Steel/Fairy types only, 1500 CP, 14 meta picks
- **Bottom nav** — Search | Leagues | Teams on all pages
- **iOS mobile support** — clipboard fallback for HTTP, 44px touch targets, active state fixes
- **Static export** — 128+ pages, all statically generated for fast loads
- **Cloudflare Pages** — auto-deploys on push, edge-cached globally

## Phase 2 — UX & Navigation (next)

Goal: make the app self-explanatory and the team building flow intuitive without a tutorial.

1. **Team building onboarding** — empty state hints on league pages so new users discover the + button flow. This is the highest-friction point right now.
2. **Expandable team panel** — mini player at bottom expands to full mode with coverage chart. Currently compact-only.
3. **Active cup awareness** — only show currently live cups, auto-rotate with GBL schedule. Removes stale content, keeps the app feeling current.
4. **Counter page type matchup visuals** — offensive/defensive chart showing why each counter works. Currently text-only.
5. **"Rate my team" entry point** — alternative path into team builder for users who don't know what to run. Lower barrier than "build a team from scratch."

---

## Phase 3 — Growth & Distribution

Goal: get the app in front of Pokemon GO players and make sharing frictionless.

1. **Shareable clean team URLs** — `/league/great-league/team/mon1-mon2-mon3` for posting in Discord/Reddit threads.
2. **SEO meta tags** — per-counter and per-league OpenGraph tags so links preview well on social.
3. **Reddit launch** — gif/video of copy-paste flow on r/TheSilphRoad, r/pokemongo, r/PokemonGOBattleLeague. Build karma first, post during US evening hours.
4. **Discord outreach** — join top Pokemon GO PvP Discords with shareable team URLs.
5. **Twitter/X clips** — short screen recordings timed to GBL season changes and raid boss rotations.
6. **Service worker / offline PWA** — cache static pages so the app works without signal (useful at GO events).

---

## Phase 4 — Data & Intelligence

Goal: make recommendations smarter and keep data fresh without manual bottlenecks.

1. **Better counter scoring** — factor in STAB, bulk, energy generation, shield pressure (currently ATK-only).
2. **Automated meta updates** — define a process for updating league JSON within 48 hours of new GBL seasons. Document steps, assign owner.
3. **Pokemon data pipeline** — pull from GO GameMaster file or PokeAPI instead of manual curation.
4. **Budget picks quarterly review** — community moves shift accessibility. Review the 46-pick list each season.
5. **Collection import** — let users import their Pokemon storage for personalized "best team from what you have" recommendations. Requires auth, storage, significant data work.

---

## Deferred

Things we are not doing yet, and why.

- **Raid counters / Rocket lineups** — these rotate frequently and need a data pipeline before they're maintainable. Revisit after Phase 4 data work.
- **Monetization** — no ads, premium, or donations until there are real users and usage data. The app is free and stays free for now.
- **Custom domain** — poke-pal.pages.dev works fine. A custom domain only matters after trademark research on the name.
- **Business plan / investor materials** — premature. Build users first.
- **Multi-language support** — English only until there's demand from non-English Pokemon GO communities.
- **Native app (iOS/Android)** — PWA covers mobile use cases. Native only if PWA limitations become a real problem.
- **ARIA/accessibility pass** — important but not blocking launch. Schedule before any major marketing push.
