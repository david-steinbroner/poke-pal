## Shipped (v0.3.7) — 2026-04-08

Everything live at https://poke-pal.pages.dev:

- **League meta** — Great League, Ultra League, Master League, Fantasy Cup with S/A/B/C tier rankings
- **Leagues landing page** (`/leagues`) — Live Now + Coming Up sections
- **Combined league search strings** — one-tap copy of all meta Pokemon names + CP cap for pasting in GO
- **Team Builder** (`/teams`) — pick 3 Pokemon for any league, see 18-type coverage chart, defensive weaknesses, meta threats, swap suggestions
- **Inline team building on league pages** — + button on every meta card, inline team section (replaced floating bar), smart suggestions
- **Fantasy Cup: Great League Edition** — Dragon/Steel/Fairy types only, 1500 CP, 14 meta picks
- **Counter page type effectiveness** — Weak to / Resists badges with 2x labels, "Build a team around X" button
- **Home page** — search-first layout, "What's Live" pill links to active cups
- **Bottom nav** — Search | Leagues | Teams on all pages, team panel positioned above it
- **S tier expanded by default**, other tiers collapsed
- **Search strings strip parenthetical form names** for GO compatibility
- **CopyBar** truncated to one line
- **"Full Analysis ->" link** when team has 2+ Pokemon
- **Version banner** on all pages (v0.3.7)
- **BackButton** with history fallback on counter pages
- **iOS mobile support** — clipboard fallback for HTTP, 44px touch targets, active state fixes
- **Static export** — 129 pages, all statically generated for fast loads
- **Cloudflare Pages** — auto-deploys on push, edge-cached globally

## Next Sprint — Pre-Launch Polish

Goal: remove the remaining blockers before sharing publicly.

1. **Persistent team state (localStorage)** — users lose their picks when navigating between pages. This is the biggest UX gap.
2. **OG tags + home page copy rewrite** — launch blocker. Link previews are dead right now (no title, no description). Home page headline needs to say what the app does.
3. **Shareable clean team URLs** (`/league/[league]/team/[mons]`) — growth engine. Users can paste team links in Discord/Reddit and recipients see the full team analysis.
4. **Role-based team analysis (lead/mid/closer)** — differentiator. No other tool tells you which Pokemon to lead with. This is the "ah ha" moment.
5. **Extract TypeBadge component + remove disabled Sentry** — code health. TypeBadge is duplicated across 6+ files. Sentry is configured but not connected.

## Deferred

Things we are not doing, and why.

- **Team building onboarding tooltips** — the design should speak for itself. If users can't figure out the + button, the design needs work, not tooltips.
- **Battle simulator** — competing with PvPoke is not our lane. Our value is search strings, not sims.
- **Monetization** — growth first. No ads, premium, or donations until there are real users.
- **Raid counters / Rocket lineups** — rotate frequently, need a data pipeline before they're maintainable.
- **Custom domain** — poke-pal.pages.dev works fine. Custom domain only matters after trademark research.
- **Native app** — PWA covers mobile use cases. Native only if PWA limitations become a real problem.
- **Multi-language support** — English only until there's demand.
