## BLOCKERS (must resolve before coding)

### P1. pvpoke rankings endpoint is wrong in spec
Engineering flagged: `pvpoke.com/data/rankings/{cup}/overall/rankings-1500.json` — real paths differ, cup slugs differ. **Fix:** manually curl the real URLs for Spring Cup + Master, hardcode in `fetch-rankings.ts`, don't template-guess.

### P2. Rankings JSON will bloat bundle ~2MB per league if imported raw
Perf + Eng agree. **Fix:**
- Post-process in scraper to `{speciesId, score, rank, fastMove, chargedMoves}` only (~50-100KB per league)
- Write to `public/data/rankings/<league>.json` (not imported — `fetch()`'d)
- Lazy-load on first ranker keystroke, not page load
- Keep autocomplete index separate

### P3. Raid tab placeholder = Reddit self-sabotage
Product's top complaint: empty tab on launch day gets roasted. **Options:**
- **(A) Cut the Raid tab entirely** — keep raids on Home, stay at 4 tabs
- **(B) Build the Raid tab properly now** — mirror Rocket page (collapsible per boss, expanded body = current raid result card). Adds maybe 3-4 hours.
- **(C) Ship placeholder** — keep current spec. Product says no.

**Recommend B.** Pattern already exists on Rocket page, data is already live (`current-raids.json`). Low-risk, high-perception win.

### P4. Leekduck scraper will likely hit Cloudflare anti-bot
Engineering flagged. **Fix:** run locally only (already the pattern), real User-Agent with contact info, 1-2s between requests, commit JSON output. Add shape-snapshot test.

## HIGH — required adjustments

### H1. Scroll-jump fix (B1) — use `onTransitionEnd` not timeout
Original spec said 200ms timeout. **Fix:** attach `onTransitionEnd` to the collapsible, call `requestAnimationFrame(() => el.scrollIntoView({block:'start'}))` after expand. Still set `scroll-margin-top: var(--sticky-header-h)`. iOS Safari sticky ancestor has known bug — fall back to `window.scrollTo({top: el.offsetTop - headerH})` if sticky breaks it.

### H2. Bottom nav on Leagues (D4) — conditional render, not "removal"
Nav is global in `layout.tsx`. **Fix:** `usePathname()` check inside `bottom-nav.tsx`, hide only on `/leagues/**`. Spec wording was ambiguous; make the implementation approach explicit.

### H3. Leagues vertical order — TABS first, then ranker
Design pushed back on spec's proposed order. **Correct order:**
1. League tabs (sticky top)
2. Ranker input
3. Meta list (compact)
4. Recommended Teams (flat section, no parent collapsible)

### H4. Drop parent "Recommended Teams" collapsible
Design flagged: 3 tap-reveal layers is too many on mobile. **Fix:** plain non-collapsible section header → team collapsibles directly under (mirror Rocket page flat pattern).

### H5. Odd-team pill layout — 3rd pill uses `col-span-2`
2-col stretched with 3 mons leaves lonely half-width cell. **Fix:** last pill `col-span-2`, reads as the team anchor.

### H6. Scraped data must be schema-validated
Security + QA: validate `{name, moveset, score, type}` shape + whitelist chars. Hand-rolled type guards, no new deps. Reject unknown fields, fail loud, preserve prior JSON on failure.

### H7. Done-definition gate upgrade
QA: current checklist too thin. **Required before marking done:**
```
npm run lint && npm run typecheck && npm test && npm run build && npm run validate-data
```
Extend `scripts/validate-data.ts` to cover new `rankings/*.json`.

### H8. Ranker UI — follow mobile keyboard best practices
Design: `inputMode="search"`, `autoCapitalize="none"`, `enterKeyHint="search"`, dropdown below input max-height 50vh, clear (×) button, result card matches meta-row visual weight (not hero).

## MEDIUM — adjustments

### M1. Ranker architecture
Split: `src/lib/ranker.ts` (lookup util) + `src/hooks/use-league-rankings.ts` (lazy-load + memo) + `src/components/league/ranker-input.tsx` (UI). Mirrors `inline-team-section` split.

### M2. Caching for scrapers
Write raw responses to `scripts/.cache/` keyed by URL+date so re-runs don't hammer sources during dev. Gitignore.

### M3. Attribution
pvpoke is GPL, leekduck has no commercial-use API. **Add:** `ATTRIBUTIONS.md`, footer credits, source links in ranker UI before Reddit.

### M4. Deactivate great-league + ultra-league cleanly
QA: stale `active: false` leagues still referenced in `curated-teams.json` + search index will cause broken links. **Add:** validator that flags orphans.

### M5. Deep-link hash validation
Security: `if (hash && /^[a-z0-9-]+$/.test(hash.slice(1)))` before `getElementById`. Also guard "fire-once" so scroll doesn't trigger when user lands without hash.

### M6. iOS clipboard fallback check
QA: `http://192.168.86.32:4000` is non-HTTPS, Clipboard API requires secure context. Verify `execCommand` fallback still in `copy-to-clipboard.ts`.

### M7. Sprite lazy-loading + dimensions
Perf: `loading="lazy" decoding="async"` + explicit width/height on all meta-list + team-pill `<img>`. `fetchpriority="low"` on collapsed teams.

### M8. Collapsible = CSS grid trick, not JS height measure
Perf: `@base-ui/react` Collapsible or `grid-template-rows: 0fr → 1fr` transition. Avoid JS-measured heights for smoother mobile animation.

### M9. Tab panel layout stability (CLS)
Perf: render all tab panels with `hidden` attr, not conditional mount. Reserve ranker dropdown space via absolute positioning.
