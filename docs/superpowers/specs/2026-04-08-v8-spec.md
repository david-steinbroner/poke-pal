## 1. New Components

### 1.1 CopyButton (`src/components/copy-button.tsx`) -- NEW

Replaces `CopyBar` entirely. No search string is displayed to the user.

```typescript
type CopyButtonProps = {
  searchString: string;
  label: string; // e.g. "Copy Search String", "Copy Meta Pokemon"
};
```

**Behavior:**
- Default state: shows `label` text
- On tap: copies `searchString` via `copyToClipboard` from `@/lib/copy-to-clipboard`
- After copy: text changes to "Copied -- paste in game!" for 3 seconds, then reverts
- Haptic feedback: `navigator.vibrate(50)` if available
- No toast notification (the button itself provides feedback)

**Styles:**
- Default: `bg-primary text-primary-foreground`
- Copied: `bg-secondary text-secondary-foreground`
- Shared: `min-h-[44px] w-full rounded-lg px-4 py-2.5 text-base font-medium transition-all active:scale-95 active:opacity-70`
- Touch: `style={{ touchAction: "manipulation" }}`
- Full-width by default; when inside `DualCopyButtons`, takes `flex-1` (half-width)

**Renders null** when `searchString` is falsy.

### 1.2 DualCopyButtons (`src/components/dual-copy-buttons.tsx`) -- NEW

Container that renders 1 or 2 `CopyButton` components side by side.

```typescript
type DualCopyButtonsProps = {
  buttons: Array<{ searchString: string; label: string }>;
};
```

**Layout:**
- Wrapper: `flex gap-2`
- Each `CopyButton` gets `className="flex-1"` applied via wrapper div
- If `buttons` array has 1 entry: single full-width button
- If `buttons` array has 2 entries: two half-width buttons side by side
- Filters out entries where `searchString` is falsy before rendering

Add `variant` prop to support both removable and addable chips.

```typescript
type PokemonChipProps = {

  onAction?: () => void;      // replaces onRemove
```

## 4. Navigation Changes

### 4.1 League Tab Memory (Feedback Item 1)

**Problem:** Tapping Leagues tab always goes to `/leagues` list, losing the user's place.

**Solution:** Two-part fix.

**Part A -- Store last league visited:**
In `LeaguePageClient`, on mount:
```typescript
useEffect(() => {
  localStorage.setItem("poke-pal:lastLeague", `/league/${leagueId}`);
}, [leagueId]);
```

**Part B -- Read on Leagues tab tap:**
In `BottomNav`, the Leagues nav item gets an `onClick` handler:
```typescript
onClick={(e) => {
  const lastLeague = localStorage.getItem("poke-pal:lastLeague");
  if (lastLeague) {
    e.preventDefault();
    router.push(lastLeague);
  }
}}
```

**Default behavior:** If no league has been visited yet, the tap navigates to `/leagues` as normal.

### 4.2 Remove Back Button from Teams Page (Feedback Items 3, 14)

In `src/app/teams/page.tsx`, delete the `<BackButton />` render and remove the `BackButton` import.

The `BackButton` component itself (`src/components/back-button.tsx`) is NOT deleted -- it is still used on counter and league pages.


**Key changes:**

**TeamSlotCard empty state update** (`src/components/team/team-slot.tsx`):

The empty slot currently renders:
```
[+ icon]
[label]
[Tap to add]
```

New empty slot with suggestions:
```
[+ icon]
[label]: Tap to add
[suggestion chips: PokemonChip variant="add" for each suggestion]
[type hint: "or any Dragon, Fairy" in text-xs text-muted-foreground]
```

This requires `TeamSlotCard` to accept new optional props:
```typescript
type TeamSlotCardProps = {
  slot: TeamSlot;
  onAdd: () => void;
  onRemove: () => void;
  label: string;
  suggestions?: Array<{ id: string; name: string }>;
  onSuggestionSelect?: (pokemonId: string) => void;
  typeHint?: string; // e.g. "or any Dragon, Fairy"
};
```

Suggestion chips render inside the dashed container, below "Tap to add", using:
```
<div className="mt-2 flex flex-wrap gap-1.5 justify-center">
  {suggestions.map(s => (
    <PokemonChip
      key={s.id}
      name={s.name}
      variant="add"
      onAction={() => onSuggestionSelect(s.id)}
    />
  ))}
</div>
{typeHint && (
  <p className="mt-1 text-xs text-muted-foreground">{typeHint}</p>
)}
```

**Suggestion logic** in `teams/page.tsx`:
- Slot 1 (Lead): always show top meta picks for the selected league
- Slot 2 (Mid): after slot 1 is filled, show `analyzeTeam().suggestions` filtered to non-selected Pokemon, plus type hint from `getGapTypes()`
- Slot 3 (Closer): after slots 1-2 are filled, same pattern with updated suggestions
